Open-Chord

This package and sub-packages represent the Open-Chord Distributed Hash Table (DHT).

Starting the Open-Chord bundle needs some system properties:

  • inria.smarttools.dht.port for setting the port of your Peer
  • inria.smarttools.dht.boot if you know other Peer you can connect to
  • inria.smarttools.dht.host if you want to specify which IP address identifies your Peer

Now, if inria.smarttools.dht.port is not specified, the port 0 will be used, and so a free port will be automatically allocated.

If inria.smarttools.dht.boot is not specified, Open-Chord will create a new Chord. It is preferable to set inria.smarttools.dht.port, so you can boot other Peer to this new Chord.

And, if inria.smarttools.dht.host is not specified, the IP address found will be used, except 127.0.0.1 because this is not an identity.

Be careful with these settings, because it is hashed. For example, if you use the port 0, the port IP:0 is actually the identity of the Peer. So you can't run multiple Chords with port 0 with the same machine. It is a bug we need to resolve at this point. You can specify both inria.smarttools.dht.port & inria.smarttools.dht.boot for resolving this issue.

The Chord is registered into OSGi as a Service. So, any of your Bundle can retrieve the Chord object with this:

final ServiceReference ref = context.getServiceReference(Chord.class.getName());
Chord chord = (Chord) context.getService(ref);
